A Run ID provider provides the JSON object which identifies a single run. In Elasticsearch each entry has this object assigned as runId.

The Default provider generates an object containing the project name, build number, Jenkins instance ID and a UID identifying the run uniquely.

Example: 

"runId": {
  "instance": "jenkins1",
  "project":  "TestPipeline",
  "build":    7,
  "uid:       "OGQyY2Q1YjMtOGE2Zi00NDYzLTkwYm_5"
}

The JSON provider generates the object based on a JSON template. This template can either be defined directly or a JSON file can be referenced.
